POO: Objeto Objetos como sabem são instâncias da classe, ou seja, após o mapeamento do

Objeto do mundo real, que em computação chamamos de Entidade, para sua representação computacional, devemos poder criar quantos objetos forem necessários. Podemos pensar nos objetos/instâncias de uma classe como variáveis do tipo definido pela classe.

NOTA: Sempre que um objeto/instância for criada deve-se passar todos os parâmetros

requeridos pelo construtor da classe (__init__), caso contrário terá um erro do tipo: TypeError

class lab35_objeto.Cliente(nome, cpf)

A classe cliente tem por objetivo prover ao sistema um padrão para o perfil de clientes a ser tratado neste sistema simples e exemplificar o uso de classes

diz()

Este método foi criado para realizar uma saudação ao cliente sempre que o mesmo acesse o sistema :return: Seu valor de retorno é a saudação

class lab35_objeto.ContaCorrente(limite, saldo, cliente)

A classe ContaCorrente estabelece um perfil de correntista para a agência bancária em questão

mostra_cliente()

Método para exibir os dados do correntista :return: retorna o nome do cliente e seu CPF

mostra_simples()

Método criado para exibir o cliente :return: Valor de retorno o correntista da conta

class lab35_objeto.Lampada(cor, voltagem, luminosidade)

O objetivo desta classe é o teste de extração de informações para documentar o Sistema A classe lâmpada de uma forma trivial permite o entendimento do processo de criação de um classe no contexto da Linguagem Python. Esta classe permite a configuração de uma lâmpada

checa_lampada()

Este método tem por objetivo verificar se uma determinada lâmpada está ligada ou não :return: seu valor de retorno será um boolean

ligar_desligar()

Esta função tem por objetivo verificar se uma função está ligada ou não e como retorno devolve um boolean :return:

class lab35_objeto.Usuario(nome, sobrenome, email, senha)

Classe usuário, permite o cadastro do cliente e lhe concede acesso ao internet banking
