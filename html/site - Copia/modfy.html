Módulo de Modelagem de novas redes

Este é um componente de apoio ao processo de desenvolvimento do Projeto FPDA no período de maio de 2021 à maio de 2023, o qual foi desenvolvido para extração de documentos de código python.

Modify

Esta classe é utilizada para modificar uma rede escolhida.
class Modfy.__init__(self, bpf_netid=None, bpf_data=None, bpf_data2=None)

Essa função inicializa os objetos da classe Modfy.

Ela requer 3 parâmetros de entrada, sendo aqui denominados bpf_netid, bpf_data e bpf_data2.

  • bpf_netid -> deve ser o nome de um arquivo json; este representa uma rede elétrica, criada a partir da classe NuNet, pertencente ao módulo de Fluxo de Potência do projeto.
bpf_data -> um dicionário que contem as informações sobre alterações feitas nos parametros de elementos já existentes na rede; suas chaves são:
    • 'bpf_equip'
    • 'bpf_name'
    • 'bpf_param'
    • 'bpf_value'
  • O valor de cada chave acima é uma lista. Com exceção da lista associada a 'bpf_value' todas as demais sãos listas de strings.
  • A lista associada a 'bpf_equip' armazena o tipo de equipamento que sofreu alteração em seus parâmetros. Os elementos dessa lista podem assumir os seguintes valores: 'bus', 'line', 'gen', 'trafo', 'trafo3w', 'load', 'shunt', 'switch', 'ext_grid'
  • A lista associada a 'bpf_name' armazena o nome do equipamento que sofreu alteração (deve ser o mesmo nome utilizado no momento em que foi criado) .
  • A lista associada a 'bpf_param' armazena strings que representam qual parâmetro do equipamento em questão foi alterado. Na documentação do pandapower estão os possíveis nome dos parâmetros de cada elemento da rede: https://pandapower.readthedocs.io/en/v2.9.0/elements.html
  • Sendo assim, as informações armazenadas em
    • bpf_data['bpf_equip'][n],
    • bpf_data['bpf_name'][n],
    • bpf_data['bpf_param'][n] e
    • bpf_data['bpf_value'][n]
    estão associadas, representando uma única modificação no parametro de um unico elemento da rede.
bpf_data2 -> um dicionario que contem as informacoes sobre alterações de elementos da rede, isto é, remoçao ou adição de um novo elemento à rede. Suas chaves são
  • 'bpf_equip', 'bpf_drop' e 'bpf_infos'
  • O valor de cada chave acima é uma lista. A lista associada a primeira chave é construída da mesma forma como vista no item 2. Ela carrega informações do tipo de elemtento removido/ adicionado.
  • A lista associada a 'bpf_drop' é toda formada por valores booleanos: True indica que o elemento foi removido; False indica que um elemento será adicionado. obrigatoria.
  • A lista associada a 'bpf_infos' é formada por dicionários. Se bpf_data2['bpf_drop'][n] == True, bpf_data2['bpf_infos'][n] será um dicionário, contendo apenas a seguinte chave:
    • 'bpf_name'
    bpf_data2['bpf_infos'][n]['bpf_name'] recebe uma string com o nome do elemento da rede que foi removido.
  • Se bpf_data2['bpf_drop'][n] == False, bpf_data2['bpf_infos'][n] também será um dicionário. Suas chaves irao depender de qual elemento foi adicionado a rede (gen, load, shunt, trafo...) e serão iguais as chaves utilizadas para construir a rede, descritas nas docstrings da classe NuNet, devendo respeitar as descricoes la feitas.
class Modfy.bpf_modfy(self)

Realiza as modificações na rede escolhida.

class Modfy.bpf_save(self)

Salva a nova rede modelada.
